Introducere

Artifacty este o aplicație Web ce realizează managementul informațiilor privitoare la colecții de artefacte arheologice. Aplicația oferă descrieri amănunțite alea artefactelor, de asemenea orice utilizator al aplicației poate adăuga artefacte noi prin intermediul unui formular si de editare a datelor cu privire la artefactele introduse de ei. Adminul având dreptul de a accepta sau de a refuza artefacte, de a importa si exporta.

Tehnologii

HTML

HyperText Markup Language (HTML) este un limbaj de marcare utilizat pentru crearea paginilor web ce pot fi afișate într-un browser (sau navigator). Scopul HTML este mai degrabă prezentarea informațiilor – paragrafe, fonturi, tabele ș.a.m.d. – decât descrierea semanticii documentului.HTML este prescurtarea de la Hyper Text Mark-up Language si este codul care sta la baza paginilor web. Va fi folosit in cadrul scheletului principal al proiectului .

CSS

CSS (Cascading Style Sheets) este un standard pentru formatarea elementelor unui document HTML. Stilurile se pot atașa elementelor HTML prin intermediul unor fișiere externe sau în cadrul documentului. CSS se poate utiliza și pentru formatarea elementelor XHTML, XML și SVGL. Intregul design al aplicatiei se va fi realizat si formatat prin intermediul CSS.

Javascript / Xmlhttprequest

JavaScript (JS) este un limbaj de programare orientat obiect bazat pe conceptul prototipurilor. Este folosit mai ales pentru introducerea unor funcționalități în paginile web, codul Javascript din aceste pagini fiind rulat de către browser. Xmlhttprequest este un obiect Javascript care permite obținerea de date în format XML (dar și HTML sau text simplu) cu ajutorul unui request HTTP. Acest obiect este utilizat în cadrul tehnologiilor Ajax . Cele doua tehnologii vor fi folosite in cadrul aplicatiei pentru validare client-side, animatii si cereri de tip AJAX.

PHP5.7

Numele PHP provine din limba engleză și este un acronim recursiv: Php: Hypertext Preprocessor. Folosit inițial pentru a produce pagini web dinamice, este folosit pe scară largă în dezvoltarea paginilor și aplicațiilor web. Se folosește în principal înglobat în codul HTML.

GitHub

Reprezinta un sistem de control al versiunilor codului sursa care va fi folosit pentru o mai buna administrare cat si pentru lucrul in echipa.

MySQL

MySQL este un sistem de gestiune a bazelor de date relaționale care va fi folosit pentru gestionarea bazei de date necesara aplicatiei web.

Arhitectura aplicatiei

Utilizand tehnologiile mentionate in sectiunea anterioara, aplicatia va fi structurata in mai multe module, dupa cum urmeaza:

1. Register / Login

Pentru a putea interacționa cu aplicația, fiecare utilizator trebuie sa se înregistreze. Formularul de înregistrare va conține date personale precum username-ul adresa de email, precum și o parola care va fi folosita pentru autentificare. În momentul în care utilizatorul trimite datele, acestea sunt verificate și, în cazul în care acestea sunt corecte, se introduc în baza de date, iar utilizatorul este redirecționat către pagina de login. În cazul în care datele nu sunt valide, userul va primi feedback și i se va cere reintroducerea unor date valide. În momentul în care utilizatorul este deja înregistrat și doreste autentificarea, va introduce username-ul și parola. Serverul preia datele, le verifică și trimite unul din două răspunsuri:

  • În cazul în care datele sunt corecte, userul va fi redirecționat către pagina principala
  • În cazul în care datele nu sunt valide, utilizatorul va primi un mesaj corespunzător și va trebui sa reintroduca datele

2. Pagina principală

După logare, utilizatorul este redirectat către pagina principală care variază în funcție de rol (administrator sau utilizator propriu-zis al aplicației). Clientul beneficiază de un meniu fix din care se poate redirecta către diferite pagini: lista de artefacte, statistici și opțiunile specifice fiecărui tip de utilizator în pagina de Profil. Un slideshow cu imagini va rula în pagina, apoi o descriere mica a aplicație îl va introduce pe utilizator în comunitatea iubitorilor de artefacte. Footerul conține informații de contact cu privire la realizatorii apicației.

3. Filtrare

Ofera posibilitatea utilizatorului de a căuta un anumit artefact un funcție de taguri, nume, an. De asemenea, utilizatorul poate folosi opțiunea de filtrare pentru a gasi anumite artefacte dupa un criteriu bine stabilit (anumiti ani, anumite locatii). Rezultatele vor fi afișate în chenarele din centrul aceleiași pagini. Pentru chenarul de cautare s-a folosit un form cu 2 chenare(introducerea datelor dupa care se realizeaza cautarea si butonul de submit). Pentru filtrare s-a folosit un select cu outgroup-uri și label-uri pentru a afișa criteriile prestabilite.

4. Profile

Pe pagina de profil a unui utilizator normal, utilizatorul va putea vedea un sumar al artefactelor introduse, fie ele acceptate, fie în curs de validare/respingere. Artefactele pot fi editate direct din aceasta pagina. De asemenea are la dispoziție un buton către pagina în care se află formularul de adăugare a unui artefact.

Pe pagina de profil adminului, adminul va putea vedea un sumar al artefactelor trimise de catre utilizatori pe care poate sa le acceptate sau să le respingă. Are la dispoziție două butoane, unul de import și unul de export, unde va putea importa și exporta fișiere de tipul CSV, JSON si XML. Apăsând pe numele artefactului se va deschide un pop-up în care se vor regăsi toate informațiile introduse. Acesta poate accepte artefacutul apasand butonul ACCEPT urmând să fie introdusă în lista de artefacte, sau să îl respingă apăsând butonul REJECT.

5. Formular de adaugare a unui artefact

In acest formular, vor fi trecute urmatoarele date:

  • Numele artefactului (Obligatoriu)
  • Anul descoperirii(începe cu o cifră, obligatoriu)
  • Regiunea
  • Muzeul în care se află sau deținătorul (Obligatoriu)
  • Descriere(Obligatoriu)
  • O imagine(Obligatoriu)

În momentul în care utilizatorul apasă pe butonul de submit, se face verificarea datelor introduse și, în cazul în care toate sunt valide. În cazul în care sunt erori, acestea vor fi afișate in locul corespunzator pe pagina formularului.

6. Pagina de descriere a unui artefact

Fiecare artefact are un meniu verical cu ancore, in pagina, pentru o deplasare mai usoara și contine toate informatiile despre acel artefact selectat anterior intr-o pagina anterioara.

7. Statistici

Păstrează formatul paginii cu formularul. În această pagină se vor realiza statistici cu privire la artefcatele pe care le vom aveam in baza de date. Am folosit butoane care deschid un meniu dropdown din care se pot alege anumite criterii pe care să se realizeze statiticile și un buton care să le genereze într-un modal box. În acel modal box va apărea o poza a un grafic ce va reprezenta statistica dorită.

Tipuri de utilizatori

1. Admin

Approve Artifact Adminul acceptă artefactele introduse de către client prin formular și le introduce în baza de date.

Reject Artifact Adminul respinge artefactele ce conțin informații nepotrivite, incomplete, irelevante.

Import Artifact(Add Artifact) Adminul adaugă în baza de date artefactele introduse de utilizatori. De asemenea poate adăuga artefacte proprii.

Export Artifact Adminul face export informațiilor din baza de date atunci când un client solicită detalii despre un anumit artefact.

View Artifacts Adminul poate vizualiza informații despre artefactele existente în baza de date.

View Statistics Adminul poate vizualiza statisticile făcute în cadrul bazei de date.

Edit Artifact Adminul poate edita informațiile despre toate artefactele aflte în baza de date.

2. Client

View Artifacts Clientul poate vizualiza detalii despre artefacte, le poate sorta și filtra după anumite criterii , de asemenea poate căuta în pagină un anumit artefact.

View Statistics Clientul poate solicita statistici ale artefactelor din baza de date.

Add Artifact Cleintul poate adăuga artefacte, completând un formular.

Edit Artifact Clientul poate modifica informațiile despre artefactele pe care le-a introdus și care se afla pe pagina lui de profil, cu unele mai actualizate, retrimițând formularul.

3. Guest

Create account Acest tip de utilizator poate sa isi creeze un cont.

Schema bazei de date

1. Tabelul ARTIFACTS

In acest tabel vor fi stocate informatiile despre fiecare artefact. Fiecare item va avea ca foreign key id-ul utilizatorului corespunzător. Astfel vom putea selecta toate artefactele unui anumit utilizator (pentru pagina de profil).

2. Tabelul USERS

Așa cum reiese din numele tabelului, USERS va conține informații despre utilizatorii aplicației. Acest tabel va fi folosit, în principal pentru autentificare (login).

Etape de dezvoltare

Aplicatia noastra web va fi dezvoltata in urmatoarele etape:

  1. Construirea partii de front-end, pe baza designului prezentat mai sus;
  2. Construirea logicii bazei de date, cu optimizarile necesare pentru o functionare cat mai rapida;
  3. Implementarea pe partea de server
  4. Testarea aplicatiei.

Bineinteles ca aceste etape se pot desfasura si concomitent, nu numai secvential.

Concluzie

În concluzie, aplicația este folosită pentru gestionarea artefactelor în funcție de preferințe, care ofera date statistice ale acestora

Pe viitor vom putea adauga noi feature-uri aplicatiei noastre, care sa imbunatateasca si mai mult experienta utilizatorilor. Aceste feature-uri ar putea fi un sistem de mesaje intre utilizatori si admin, o sortare a artefactelor sau optiunea de editare profil.
